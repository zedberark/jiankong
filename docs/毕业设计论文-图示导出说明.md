# 毕业设计论文 — 图示文件与导出说明

本文档说明 **NetPulse** 论文插图的**集中存放位置**与**导出为 Word 可用图片**的步骤。图号（如图 3-1）可按学校模板在 Word 中统一调整，本仓库中的编号仅为**建议**（对应常见「第 3 章总体设计、第 4 章数据库设计」结构）。

---

## 一、文件位置（论文优先使用）

| 类型 | 路径 | 说明 |
|------|------|------|
| **draw.io 全量（推荐排版）** | `docs/NetPulse-图示全量.drawio` | 单文件 **6 个页签**：总体架构、功能架构、侧栏模块、后端分层类图、全局 E-R、系统功能全流程（登录）。 |
| **PlantUML 源（矢量、易改字）** | `docs/NetPulse-图示全量-UML.puml` | 与上表对应，含 **图 3-1～图 4-1** 等题注；可改章节号后导出。 |
| **合并脚本（需重新生成全量 draw.io 时）** | `docs/tools/merge_netpulse_drawio.py` | 从分散的 `NetPulse-系统架构图.drawio` 等源文件合并；在项目根执行：`python docs/tools/merge_netpulse_drawio.py` |

分散的原始图仍保留在 `docs/` 下（如 `NetPulse-功能流程图-完整版.drawio`），便于单独修改某一类图后再运行合并脚本。

---

## 二、draw.io（diagrams.net）导出

1. 用浏览器打开 [https://app.diagrams.net/](https://app.diagrams.net/)，**文件 → 打开**，选择 `NetPulse-图示全量.drawio`。
2. 底部切换**页签**，逐页导出：
   - **文件 → 导出为 → PNG**（建议缩放 **200%** 或更高，保证打印清晰）；
   - 或 **导出为 → SVG**（Word 2016+ 可直接插入，放大不失真）。
3. 论文中建议图题格式：**图 3-1 系统总体架构图**（与 PlantUML 文件内标题一致时可复制）。

---

## 三、PlantUML 导出

1. 安装 **VS Code** 扩展 **PlantUML**（需本机 **Java**；或改用在线渲染）。
2. 打开 `NetPulse-图示全量-UML.puml`，每个 `@startuml`～`@enduml` 为**一张独立图**，在预览中 **右键 → Export Current Diagram** 导出 PNG/SVG。
3. 在线方式：将某一图块复制到 [PlantUML Web Server](https://www.plantuml.com/plantuml/uml)（注意一次只粘贴**一个** `@startuml` 块）。

## 五、重新生成「全量 draw.io」

修改任意源 `.drawio` 后，在项目目录执行：

```bash
python docs/tools/merge_netpulse_drawio.py
```

将覆盖生成 `docs/NetPulse-图示全量.drawio`。
