## 摘要

随着企业信息化与网络规模的不断扩大，不论在业务保障还是在运维决策中，网络与设备状态信息的重要性都是毋庸置疑的。虽然这些信息以数据的形式存在于系统之中，但其价值却是可量化的——只有做到对设备状态与性能指标**更好、更快、更准确**的采集与展示，才能支撑运维人员做出正确的判断，保障业务连续性。运维信息通过集中监控平台汇聚与发布，让不同岗位的人员打破空间的界限，实时、同步掌握全网设备健康与告警情况，使团队内部获取运维信息的途径差异大大减少，同时也可以随时随地进行监控查看与远程运维操作。

在 Linux 及企业运维领域，**Zabbix** 等成熟监控产品已成为日常监控的主流选择；本人以 **Zabbix 作为日常监控平台**，在实践中形成了对集中监控与运维自动化的需求认识，**在此基础上基于 Spring Boot** 进行架构设计和技术选型，设计并实现了一款**网络监控与运维平台 NetPulse**。基于 B/S 架构的系统开发与设计，结合监控运维平台的具体业务要求，采用 **Java 语言**与 **Vue** 技术，将**前端与后端分离**，便于代码维护和拓展，同时可以更好地处理业务逻辑；使用 **Vue 与 ECharts** 实现前端页面展示和用户交互，通过 **RESTful 接口**与 **WebSocket** 实现前后端业务逻辑处理，并依托 **MySQL、Redis、InfluxDB** 等存储与缓存技术完成数据持久化与实时指标展示，从而完成系统的核心功能。

本论文完成了基于 **Spring Boot** 与 **Vue** 的网络监控与运维平台 NetPulse 的设计与开发，提供了**设备统一管理、实时指标监控、告警规则与历史、Web SSH 远程终端、配置备份与恢复、操作审计、用户与权限管理**以及**个性化 AI 运维助手**等功能，满足运维人员对集中监控与高效运维的需求，更好地服务于企业网络与信息系统的管理。

**关键词**：Spring Boot；运维监控平台；Vue；前后端分离；MySQL  
（题目：基于 Spring Boot 的运维监控平台设计与实现；关键词 3～5 个见 `论文题目与关键词.md`）
