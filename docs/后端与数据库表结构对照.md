# 后端实体与 MySQL 表结构对照（便于对比现有库是否少/多字段）

以下列出**后端 JPA 实体期望的库表及列名**。请用 `SHOW CREATE TABLE 表名;` 或 `DESCRIBE 表名;` 与下表逐表对比，看是否多列或少列。

**说明**：
- 列名以**数据库列名**为准（`@Column(name="xxx")` 的 `xxx`，未写则与字段名一致）。
- 若你当前库是由早期脚本或 JPA `ddl-auto` 建的，可能多出已被迁移删掉的列，或少掉实体里新增的列；按本表 + 下方「迁移变更摘要」逐项核对即可。

## 1. alert_rule（告警规则）

| 列名 | 类型/说明 | 实体字段 | 注释 |
|------|-----------|----------|------|
| id | BIGINT PK AUTO_INCREMENT | id | 主键 |
| name | VARCHAR(100) NOT NULL | name | 规则名称 |
| device_id | BIGINT | deviceId | 指定设备（空表示按设备类型匹配） |
| device_types | VARCHAR(64) | deviceTypes | 设备类型过滤，逗号分隔如 server,firewall |
| metric_key | VARCHAR(50) NOT NULL | metricKey | 指标键，如 cpu_usage |
| rule_condition | VARCHAR(255) NOT NULL | condition | 告警条件表达式，如 > 80 |
| severity | ENUM('info','warning','critical') | severity | 严重程度 |
| enabled | BOOLEAN/TINYINT NOT NULL | enabled | 是否启用 |
| notify_email | BOOLEAN/TINYINT | notifyEmail | 触发时是否发送邮件（V13 新增） |
| auto_fix_enabled | BOOLEAN/TINYINT | autoFixEnabled | 是否启用告警自动修复 |
| auto_fix_type | VARCHAR(32) | autoFixType | 修复方式：ssh_command / local_script |
| auto_fix_command | TEXT | autoFixCommand | 修复命令或脚本路径 |
| create_time | DATETIME NOT NULL | createTime | 创建时间 |
| update_time | DATETIME NOT NULL | updateTime | 更新时间 |

**迁移**：V10 会执行 `DROP COLUMN notify_channels`、`DROP COLUMN duration`。若某列不存在会报错，可注释 V10 中对应语句后执行。

**不应存在的列**（若存在说明未跑完迁移或旧脚本）：`notify_channels`、`monitor_item_id`（V6 已删）、`duration`（后端未使用，V10 已删）。

## 迁移变更摘要（按版本）

| 版本 | 操作 |
|------|------|
| V2 | 建 system_config、config_backup（无 summary 列） |
| V3 | alert_rule：无；sys_user：DROP COLUMN real_name |
| V4 | audit_log：DROP FK fk_audit_user，DROP COLUMN user_id；monitor_item：DROP oid, unit |
| V5 | DROP role_permission、sys_permission |
| V6 | alert_rule DROP FK fk_alert_rule_monitor、DROP monitor_item_id；alert_history DROP monitor_item_id；DROP monitor_item |
| V7 | sys_user DROP COLUMN phone |
| V8 | DROP webssh_session |
| V9 | DROP notification_log |
| V10 | alert_rule DROP COLUMN notify_channels、duration |
| V11 | 建 role_menu（角色权限模板） |
| V12 | 安全清理：仅当存在时删除未使用列与表（同上），可重复执行 |
| V14 | DROP TABLE user_menu（菜单权限统一按 role_menu） |
| V15 | 再次安全清理不需要的列与表（与 V12 一致，可重复执行） |

## 快速对比方法

在 MySQL 中执行（把 `表名` 换成上面各表名）：

```sql
DESCRIBE 表名;
-- 或
SHOW COLUMNS FROM 表名;
```

把结果列名与上表「列名」对比：  
- **多出来的列**：看是否在上表「不应存在的列」或迁移 DROP 列表中，若是则可按迁移补执行或保留不删（若不影响运行）。  
- **少了的列**：按上表类型用 `ALTER TABLE ... ADD COLUMN ...` 补上，或执行/补充 Flyway 迁移。

若需要，我可以再按你当前库的 `SHOW CREATE TABLE` 输出帮你逐表写出一份「差异与修补 SQL」。
